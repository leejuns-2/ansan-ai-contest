# 데이터 인벤토리 v1

## 작성 원칙

- 실제 제공 여부가 확인된 공개 데이터와, 안산시 내부 확인이 필요한 데이터를 구분한다.
- 공공데이터 활용 심사에서 설명 가능한 수준으로 `제공기관`, `예상 필드`, `활용 목적`, `확보 리스크`, `대체 데이터`를 명시한다.
- 최종 제출 전에는 각 데이터의 최신 등록일, 필드, API 신청 필요 여부, 안산시/대부도 필터 가능 여부를 다시 확인한다.

## 1단계 MVP 최소 데이터셋

1단계는 공개 데이터 중심으로 설계한다.

| 필수 여부 | 데이터 | 사용 방식 |
|---|---|---|
| 필수 | 조위 실측·예측 | 만조까지 남은 시간, 조위 상승 속도 |
| 필수 | 기상 예보·실황 | 풍속, 강수, 기상악화 위험 |
| 필수 | 연안사고 이력 | 장소 유형별 사고이력 점수 |
| 필수 후보 | 유동인구 집계 | 시간대별 방문 집중도 |
| 필수 | 공간정보/POI | 위험지점, 시설, 주차장, 관광지 거리 |
| 선택 | 현장 비식별 신호 | 2단계 시범구간에서 정확도 보강 |

## 표준 피처 매핑

| 모델 피처 | 원천 데이터 | 산출 방식 |
|---|---|---|
| `minutes_to_high_tide` | 조위 실측·예측 | 예측 만조 시각과 현재 시각 차이 |
| `tide_change_speed` | 조위 실측·예측 | 직전 시간 대비 조위 변화량 |
| `wind_risk` | 기상청 예보·실황 | 풍속 구간별 점수화 |
| `rain_risk` | 기상청 예보·실황 | 강수량 또는 강수 여부 점수화 |
| `visitor_index` | 유동인구/방문소비 | 평소 동일 시간대 대비 비율 |
| `accident_history_score` | 연안사고 이력 | 장소 유형·시간대·원인별 가중치 |
| `spatial_risk_score` | 공간정보 | 갯벌·방파제·해안 접근부 여부, 안전시설 거리 |
| `anonymous_stay_signal` | 현장 비식별 신호 | 체류시간 증가, 위험구역 방향 이동량 |

## 데이터 확보 리스크 대응

| 리스크 | 대응 |
|---|---|
| 안산 세부 사고 데이터가 부족함 | 장소명·해경서·발생지역 필터를 검증하고, 부족하면 장소 유형 기반 위험도와 해경 예측분석모델을 병행한다. |
| 유동인구 해상도가 낮음 | 1단계는 시군구/행정동 단위 방문 집중도로 설명하고, 2단계에서 시범구간 현장 신호를 보강한다. |
| 조위관측소가 위험지점과 떨어져 있음 | 관측소 위치와 위험지점 거리, 대표성을 검토하고 인근 관측소 또는 예측 조위 자료를 병행한다. |
| 현장 영상 데이터 사용이 민감함 | 원본 영상 저장을 전제로 하지 않고 엣지 AI 비식별 집계만 사용한다. 얼굴인식·개인추적은 제외한다. |
| 행정 조치 이력이 구조화되어 있지 않음 | MVP에서는 담당자가 선택한 추천 카드와 실행 시간을 로그로 저장하는 방식으로 시작한다. |
