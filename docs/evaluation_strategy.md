## 득점 전략 요약

| 심사항목 | 보여줘야 할 것 | 현재 아이디어의 대응 |
|---|---|---|
| 공공데이터 활용 | 여러 공공데이터의 결합 필요성 | 조위·기상·연안사고·유동인구·공간정보 결합 |
| AI 활용 | AI가 없으면 어려운 판단 | 시간·장소·상황 조합 기반 위험도 예측 및 개입 추천 |
| 기획력 | 문제 정의와 행정 실행 흐름 | 사후 대응이 아닌 사전 예방 행정으로 구조화 |
| 효과성 | 시민 안전과 행정 효율 | 위험 노출 감소, 인력·시설 배치 최적화 |
| 창의성 | 흔한 앱/챗봇과 차별 | 익명 상황인식 + 예방 개입 추천 + 효과 피드백 |
| 실현가능성 | 단계적 MVP | 공개데이터 기반 MVP → 센서 연동 → 고도화 |
| 발표력 | 한눈에 이해되는 데모 | 위험지도, 위험 원인, 추천 개입 카드, 효과 리포트 |

## 심사위원이 좋아할 포인트

### 1. 안산시 특수성

안산은 연안·도서·관광·산업·도시 기능이 결합된 지자체다. 단순 민원 챗봇보다 “연안 관광 안전”은 지역성이 강하다.

단, 특정 장소와 사고 사례는 반드시 공식 자료로 추가 검증해야 한다.

### 2. 공공데이터 결합의 필연성

위험은 조위 하나로 판단되지 않는다. 조위, 기상, 방문객, 시간대, 과거 사고, 공간 구조가 겹칠 때 커진다. 이 복합성을 강조하면 공공데이터 활용 점수가 올라간다.

### 3. AI 활용의 자연스러움

AI는 단순 질의응답이 아니라 다음을 수행한다.

- 시공간 위험도 예측
- 이상 혼잡 탐지
- 익명 행동 패턴 인식
- 개입 추천
- 효과 피드백

### 4. 실현가능성

처음부터 거대한 통합관제센터를 만들겠다고 하면 위험하다. 다음처럼 단계화한다.

1. 공개데이터 기반 위험지도 MVP
2. 시범구간 현장 신호 연동
3. 관계기관 공유 체계 구축
4. 효과 검증 후 확산

### 5. 정책 확장성

안산 연안에서 시작하되, 다음으로 확장 가능하다.

- 다른 연안 도시
- 하천·호수 관광지
- 산책로·둘레길 안전
- 폭염·침수·축제 혼잡 예방

## 심사위원이 싫어할 포인트와 방어 논리

### 우려 1. “해경 업무 아닌가?”

방어:

- 구조·통제·단속이 아니라 예방 행정 지원이다.
- 안산시는 안내, 시설, 관광, 교통, 관계기관 공유를 담당한다.
- 권한 주체를 대시보드에서 구분한다.

### 우려 2. “CCTV 감시 아닌가?”

방어:

- 얼굴인식 없음.
- 개인 추적 없음.
- 원본 영상 저장 없음 또는 최소화.
- 엣지에서 익명 카운팅·방향·체류시간만 추출.
- 개인이 아니라 장소·시간 단위 개입.

### 우려 3. “데이터가 있나?”

방어:

- 공개 공공데이터로 1단계 위험지도 구현 가능.
- 현장 신호는 시범구간부터 단계적 연동.
- 데이터 부족 구간은 규칙 기반 + 공무원 피드백으로 보정.

### 우려 4. “AI가 꼭 필요한가?”

방어:

- 조위, 기상, 방문객, 사고이력, 공간구조가 동시에 변하므로 사람이 매일 수동 판단하기 어렵다.
- AI는 위험 조합을 자동으로 계산하고, 원인과 개입을 설명한다.

### 우려 5. “효과를 어떻게 입증하나?”

방어:

- 시범구간 전후 비교
- 위험구역 체류시간 변화
- 신고/민원 변화
- 고위험 시간대 잔류 인원 변화
- 시설 개선 전후 위험도 변화

## 발표 구조 추천

### 1분: 문제 제기

“연안 사고는 사고가 난 뒤 빨리 대응하는 것도 중요하지만, 더 좋은 행정은 위험이 만들어지기 전에 개입하는 것입니다.”

### 2분: 안산시 맥락

대부도 등 연안 관광·낚시·갯벌 활동 공간의 특성 설명. 단, 수치와 장소는 공식 자료로 보강.

### 3분: 서비스 개념

위험 예측 → 익명 상황인식 → 개입 추천 → 효과 피드백.

### 3분: 데이터·AI

조위, 기상, 유동인구, 사고이력, 공간정보를 어떻게 결합하는지 보여준다.

### 3분: MVP 데모

지도, 위험 원인, 추천 카드, 공유 리포트, 효과 리포트 순서.

### 2분: 기대효과와 확산성

안전, 행정 효율, 관광 이미지, 타 지자체 확산.

### 1분: 권한·개인정보 리스크 대응

해경 권한 대체 아님. 익명·집계 기반.
