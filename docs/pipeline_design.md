# 파이프라인 설계 v0

## 설계 목표

`안산 세이프코스트 AI`의 파이프라인은 시민용 앱이 아니라, 안산시 담당자가 연안 위험을 미리 보고 예방 조치를 선택하도록 돕는 행정 의사결정 흐름이다.

핵심 판단은 다음 한 문장으로 정리한다.

> 조위·기상·유동인구·사고이력·공간정보·익명 현장 신호를 장소-시간 단위로 결합해 고위험상황을 예측하고, 안산시가 수행 가능한 예방 행정 조치를 추천한다.

## 전체 흐름

```text
공공·행정·현장 데이터 수집
        ↓
장소-시간 표준화
        ↓
위험 피처 생성
        ↓
AI 위험도 예측 및 이상상황 탐지
        ↓
위험 원인 설명
        ↓
예방 개입 추천
        ↓
담당자 검토 및 실행
        ↓
관계기관 공유·시민 안내·시설 개선
        ↓
개입 효과 분석 및 모델/정책 피드백
```

## 1. 데이터 수집 파이프라인

### 입력 데이터

| 범주 | 후보 데이터 | 1단계 MVP 사용 | 비고 |
|---|---|---:|---|
| 조위·해양 | 국립해양조사원 조위 실측·예측 | 필수 | 안산 인근 관측소 적합성 검증 필요 |
| 기상 | 기상청 단기예보·실황·특보 | 필수 | 풍속, 강수, 시정, 기온 중심 |
| 연안사고 | 해양경찰청 연안사고 이력 | 필수 | 안산/대부도 필터 가능 여부 검증 필요 |
| 유동인구 | 안산시/경기도 유동인구, 방문·소비 데이터 | 필수 후보 | 시간·공간 해상도 검증 필요 |
| 공간정보 | 해안, 갯벌, 방파제, 항구, 주차장, 관광지, 안전시설 | 필수 | 공개데이터와 내부 행정 데이터 병행 |
| 현장 신호 | CCTV/IoT 익명 카운팅, 체류시간, 이동 방향 | 2단계 | 얼굴인식·개인추적 제외 |
| 행정 이력 | 전광판 문구, 안내방송, 안전요원 배치, 시설 개선 이력 | 2단계 | 효과 분석용 |

### 수집 원칙

- 공개 공공데이터로 1단계 MVP를 구성한다.
- 실제 제공 여부가 불확실한 데이터는 `검증 필요`로 표시한다.
- 현장 영상은 원본 저장을 전제로 하지 않고, 엣지 단말에서 추출한 비식별 집계 신호만 사용한다.
- 개인 단위 데이터가 아니라 `장소-시간 단위` 데이터셋으로 통합한다.

## 2. 표준화 파이프라인

모든 데이터는 `장소 단위`와 `시간 단위`를 맞춰 결합한다.

### 장소 단위

1단계 MVP에서는 세밀한 좌표 기반보다 설명 가능한 위험지점 POI 방식을 우선한다.

예시:

| location_id | 장소 유형 | 예시 명칭 | 비고 |
|---|---|---|---|
| A01 | 갯벌·해안 접근부 | 방아머리 후보 구역 | 실제 위험 근거 검증 필요 |
| A02 | 항구·방파제 | 탄도항 후보 구역 | 관리 주체 검증 필요 |
| A03 | 시화호/방조제 인근 | 시화방조제 후보 구역 | 관할·위험사례 검증 필요 |

고도화 단계에서는 POI를 격자 또는 세부 위험구역 단위로 확장한다.

### 시간 단위

- MVP 기본 단위: 1시간
- 현장 신호 연동 후: 10분 또는 15분
- 예측 범위: 당일 0~24시간, 발표 데모에서는 주말 오후 고위험 시간대 중심

### 표준 테이블

```text
datetime
location_id
location_type
tide_level
tide_phase
minutes_to_high_tide
tide_change_speed
wind_speed
rainfall
weather_alert_flag
visitor_index
visitor_change_rate
accident_history_score
spatial_risk_score
anonymous_crowd_count
anonymous_stay_duration
intervention_history
risk_score
risk_level
```

## 3. 피처 생성 파이프라인

### 해양·기상 위험 피처

- 만조까지 남은 시간
- 간조/만조 전후 여부
- 조위 상승 속도
- 풍속 구간
- 강수 여부
- 기상특보 여부
- 일몰 전후 여부

### 방문·혼잡 피처

- 시간대별 유동인구 지수
- 평소 대비 혼잡도
- 전 시간 대비 증가율
- 주말·공휴일·성수기 여부
- 주차장/관광지 인근 집중도

### 공간 위험 피처

- 갯벌, 방파제, 해안 접근부 여부
- 안전시설 존재 여부
- 안내판/조명/펜스 보강 필요 여부
- 과거 유사 사고 이력
- 관계기관 관리구역 여부

### 익명 상황인식 피처

2단계부터 적용한다.

- 구역별 추정 인원 수
- 위험구역 방향 이동량
- 체류시간 증가
- 복귀 지연 추정 군집
- 야간 또는 강풍 시간대 해안 근접 체류

사용하지 않는 피처:

- 얼굴 특징
- 차량번호
- 개인별 이동경로
- 개인별 위험점수
- 특정 개인 식별자

## 4. AI 분석 파이프라인

### 4.1 위험도 예측

목표는 `장소-시간별 위험도`를 산출하는 것이다.

MVP 접근:

- 규칙 기반 위험 산식 + 설명 가능한 ML 모델 후보
- 모델 후보: 로지스틱 회귀, 랜덤포레스트, Gradient Boosting
- 출력: 위험도 0~100점, 낮음/주의/높음/긴급 등급

고도화 접근:

- 시계열 예측
- 시공간 그래프 모델
- 위험구역 간 연쇄 혼잡 분석

### 4.2 이상 혼잡 탐지

목표는 평소와 다른 방문·체류 패턴을 감지하는 것이다.

MVP 접근:

- 이동평균 대비 편차
- 전주/전월 동일 시간대 대비 증가율
- 임계치 기반 이상 탐지

고도화 접근:

- Isolation Forest
- 시계열 이상탐지
- 군집 밀도 변화 탐지

### 4.3 위험 원인 설명

담당자와 심사위원이 AI 판단 근거를 볼 수 있어야 한다.

예시 출력:

```text
위험도 82점 / 높음
주요 원인:
1. 만조까지 47분
2. 유동인구 평소 대비 165%
3. 풍속 상승
4. 일몰 시간대 진입
5. 과거 유사 조건 사고 이력
```

### 4.4 예방 개입 추천

위험도만 표시하면 점수판에 그치므로, 행정 조치 추천까지 연결한다.

추천 로직 v0:

```text
위험 유형 = 조위 위험 + 혼잡 위험 + 공간 위험 + 현장 신호
        ↓
개입 후보 매칭
        ↓
권한 주체 필터링
        ↓
실행 난이도·예상 효과 정렬
        ↓
담당자에게 추천 카드 표시
```

권한 필터링 원칙:

- 안산시 직접 수행 가능: 안내, 전광판, 관광안내, 시설 개선, 관계기관 공유
- 관계기관 협업 필요: 해경·소방·경찰 상황 공유, 순찰 협조 요청
- 제외: 출입통제 자동화, 구조 지휘, 강제 통제, 개인 제재

## 5. 운영 의사결정 파이프라인

### 담당자 화면 흐름

1. 오늘의 연안 위험지도를 확인한다.
2. 고위험 구간을 클릭해 위험 원인을 본다.
3. 추천 개입 카드를 검토한다.
4. 실행 가능한 조치를 선택한다.
5. 관계기관 공유 리포트를 생성한다.
6. 실행 이력을 저장한다.
7. 다음 날 또는 다음 주 효과 리포트를 확인한다.

### 개입 유형

| 위험상황 | 추천 개입 | 실행 주체 | 권한 리스크 |
|---|---|---|---|
| 조위 상승 + 갯벌 방향 이동 증가 | 전광판·안내방송·관광안내 채널 복귀 안내 | 안산시 | 낮음 |
| 강풍 + 방파제 체류 증가 | 해안 접근 주의 안내, 순찰 협조 요청 | 안산시 + 관계기관 | 낮음 |
| 주말 혼잡 + 일몰 전후 체류 증가 | 사전 알림, 안전요원 배치 권고 | 안산시 | 낮음 |
| 반복 위험구간 발생 | 안내판 위치 개선, 조명 보강, 동선 유도 시설 후보 등록 | 안산시 | 낮음 |
| 구조·출입통제 필요 가능성 | 위험 예측 리포트 관계기관 공유 | 해경 등 관계기관 | 안산시 직접 통제 금지 |

## 6. 피드백 파이프라인

개입 효과는 사고 건수만으로 평가하지 않는다. 사고는 드문 이벤트라 단기 검증이 어렵기 때문이다.

### 선행 KPI

- 고위험 시간대 위험구역 체류 추정 인원 변화
- 안내 후 복귀율 또는 체류시간 변화
- 위험구역 접근량 변화
- 신고·민원 변화
- 관계기관 사전 공유 건수
- 시설 개선 전후 위험도 변화
- 담당자 조치 소요시간 변화

### 학습 루프

```text
추천 조치 실행
        ↓
실행 시각·장소·채널 기록
        ↓
개입 전후 지표 비교
        ↓
효과 높은 조치 가중치 상향
        ↓
반복 위험구간 정책 리포트 생성
```

## 7. MVP 구현 순서

### 1단계: 문서형 데모 데이터셋

- 시범 장소 3~5곳 정의
- 1시간 단위 샘플 데이터 생성
- 조위·기상·유동인구·사고이력 피처 예시 구성
- 위험도 산식과 원인 Top 3 생성

산출물:

- `data_inventory.md`
- 샘플 데이터 표
- 위험도 산식 설명

### 2단계: 대시보드 와이어프레임

- 오늘의 위험지도
- 위험도 상세 패널
- 예방 개입 추천 카드
- 관계기관 공유 리포트
- 개입 효과 리포트

산출물:

- `mvp_wireframes.md`
- 발표용 화면 설명

### 3단계: 제출서 반영

- 공공데이터 활용 구조
- AI 모델 구조
- 개인정보 보호 설계
- 해경 권한 경계
- 단계별 추진계획

산출물:

- `submission_draft_v1.md`
- `presentation_outline.md`
- `judge_qna.md`

## 8. 최종 제안서에 넣을 파이프라인 요약

```text
1. 공공데이터 수집
   조위·기상·사고이력·유동인구·공간정보를 수집한다.

2. 장소-시간 결합
   데이터를 위험지점과 시간대 기준으로 통합한다.

3. AI 위험도 예측
   위험도 점수와 위험 상승 원인을 산출한다.

4. 익명 상황인식
   시범구간에서는 사람 수·방향·체류시간 등 비식별 집계 신호만 활용한다.

5. 예방 개입 추천
   전광판, 안내방송, 관광안내, 시설 개선, 관계기관 공유 등 안산시가 수행 가능한 조치를 추천한다.

6. 효과 피드백
   개입 전후 변화를 분석해 다음 위험 예측과 정책 개선에 반영한다.
```

## 9. 설계상 주의사항

- `고위험자` 대신 `고위험상황`, `고위험 군집·행동 패턴`을 사용한다.
- `CCTV 감시` 대신 `익명 상황인식`, `비식별 집계 신호`를 사용한다.
- 안산시가 출입통제·구조지휘를 하는 것처럼 쓰지 않는다.
- 데이터 제공 여부와 사고 통계는 최종 제출 전 공식 출처로 검증한다.
- MVP는 공개데이터 기반으로 충분히 설명하고, 현장 센서 연동은 단계적 확장으로 둔다.
