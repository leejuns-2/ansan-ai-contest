# 데이터·AI 활용 계획

## 설계 원칙

이 공모전의 핵심 점수는 공공데이터 활용과 AI 활용이다. 따라서 제출서에서는 “AI를 붙인 앱”이 아니라, **공공데이터를 결합해야만 풀 수 있는 문제**로 설계해야 한다.

## 데이터 후보

아래 데이터는 실제 제출 전 최신 제공 여부, API 신청 필요 여부, 필드 구성, 갱신 주기를 다시 확인해야 한다.

| 데이터 범주 | 후보 데이터 | 활용 목적 | 비고 |
|---|---|---|---|
| 연안사고 | 해양경찰청 연안사고 이력, 원인별 연안사고 현황 | 사고 유형·시간·장소·원인 패턴 학습 | 안산/대부도 필터 가능 여부 확인 필요 |
| 조위/해양 | 국립해양조사원 조위관측소 실측·예측 조위, 조위관측소 운영 현황 | 물때·조위 변화 기반 위험도 예측 | 가장 중요한 차별 데이터 |
| 기상 | 기상청 단기예보·실황, 강수·풍속·기온 등 | 강풍·강수·시정 악화 위험 반영 | API 필드 확인 필요 |
| 유동인구 | 안산시 시간대별 유동인구, 경기데이터드림 방문·소비 데이터 | 관광객 집중, 고위험 시간대 추정 | 격자/행정동 단위 해상도 확인 필요 |
| 공간정보 | 해안선, 도로, 주차장, 관광지, 안전시설 위치 | 위험지도, 동선 분석 | 안산시 내부 데이터가 있으면 강화 |
| 교통/주차 | 주차장, 도로 혼잡, 대중교통 | 방문 집중 예측 보조 | 민간/공공 결합 가능 |
| 현장 신호 | CCTV 익명 카운팅, IoT 인원 센서, 전광판 운영 이력 | 실시간 위험상황 인식 | MVP에서는 선택적 확장으로 제시 |
| 행정 데이터 | 민원, 안전조치 이력, 시설물 개선 이력 | 정책 피드백·효과 분석 | 내부 데이터 필요 |

## 데이터 결합 구조

```text
장소 단위: 격자 또는 위험지점 POI
시간 단위: 10분/1시간 단위

[조위] + [기상] + [유동인구] + [요일/계절] + [사고이력] + [공간 특성]
       ↓
장소-시간별 위험도 피처 테이블
       ↓
AI 위험도 예측 및 개입 추천
```

## 핵심 피처 예시

### 시간 피처

- 시간대
- 요일
- 공휴일 여부
- 성수기/비성수기
- 일몰 전후 여부
- 야간 여부

### 해양·기상 피처

- 조위 실측값
- 예측 조위
- 조위 상승/하강 속도
- 만조/간조까지 남은 시간
- 풍속
- 강수량
- 기온
- 기상특보 여부

### 공간 피처

- 해안/방파제/갯벌/항구/주차장과의 거리
- 위험구역 여부
- 안전시설 존재 여부
- 안내판/조명/펜스 존재 여부
- 과거 사고 발생 여부

### 방문·혼잡 피처

- 시간대별 유동인구
- 전 시간 대비 증가율
- 평소 대비 혼잡도
- 특정 방향 이동량
- 체류 시간 추정

## AI 모델 설계

### 1. 시공간 위험도 예측 모델

목표:

- 장소·시간별 위험도 점수 산출

가능한 접근:

- 초기 MVP: 규칙 기반 + 로지스틱 회귀/랜덤포레스트/Gradient Boosting
- 고도화: 시계열 모델, Graph Neural Network, Spatio-temporal Transformer

MVP에서는 복잡한 모델명을 과시하기보다, “설명 가능한 위험도 모델”이 유리하다.

출력:

- 위험도 0~100점
- 낮음/주의/높음/긴급 등급
- 위험 상승 원인 Top 3

### 2. 이상 혼잡 탐지 모델

목표:

- 평소와 다른 방문·체류 패턴 탐지

가능한 접근:

- 이동평균 대비 편차
- Isolation Forest
- 시계열 이상탐지
- 군집 밀도 변화 탐지

출력:

- 평소 대비 혼잡 증가율
- 특정 지점 체류 증가 알림
- 위험시간과 겹칠 때 우선 경보

### 3. 익명 위험행동 패턴 탐지

목표:

- 개인 식별 없이 위험구역 접근, 체류, 복귀 지연 패턴을 감지

가능한 접근:

- CCTV/센서에서 사람 수·방향·체류시간만 추출
- 원본 영상 저장 금지 또는 최소화
- 얼굴/개인식별정보 비수집
- 엣지 단말에서 좌표·카운트만 전송

출력:

- 위험구역 방향 이동량
- 잔류 추정 군집 수
- 체류 시간 증가

### 4. 예방 개입 추천 모델

목표:

- 위험 유형별로 안산시가 할 수 있는 조치를 추천

가능한 접근:

- 규칙 기반 추천으로 시작
- 과거 개입 효과 데이터를 누적해 Contextual Bandit 또는 강화학습으로 고도화

추천 예시:

| 상황 | 추천 개입 |
|---|---|
| 조위 상승 + 갯벌 방향 이동 증가 | 갯벌 복귀 안내방송, 전광판 경고, 관광안내 채널 알림 |
| 강풍 + 방파제 체류 증가 | 해안 접근 주의 안내, 현장 순찰 협조 요청 |
| 주말 혼잡 + 일몰 전후 체류 증가 | 사전 알림, 안전요원 배치 권고, 주차장 안내 강화 |
| 반복 위험구역 발생 | 시설 개선 후보 등록, 안내판 위치 개선, 조명 보강 제안 |

## 설명가능성 설계

심사위원과 현장 공무원은 “AI가 왜 이렇게 판단했는지”를 봐야 한다.

대시보드에는 다음 설명을 표시한다.

```text
위험도: 82점 / 높음
주요 원인:
1. 만조까지 47분
2. 평소 대비 유동인구 165%
3. 풍속 상승
4. 과거 유사 시간대 사고 이력 존재
추천 조치:
- 전광판 문구를 '갯벌 체류 주의'로 전환
- 관광안내 채널에 17:00 이전 복귀 안내 발송
- 관계기관에 고위험 시간대 리포트 공유
```

## 개인정보 보호 설계

반드시 제안서에 포함할 내용:

- 얼굴인식 미사용
- 개인별 이동경로 저장 금지
- 원본 영상 미저장 또는 최소 저장
- 엣지 AI로 사람 수·방향·체류시간만 추출
- 격자/구역 단위 집계
- 개인 대상 제재가 아니라 장소·시간 단위 예방 개입

## MVP 데이터 전략

실제 데이터 확보가 늦어질 경우에도 제출서와 발표에서 설득 가능한 방식:

1. 공공데이터포털/경기데이터드림의 공개 데이터 구조를 바탕으로 샘플 데이터셋 생성
2. 대부도 등 특정 시범구간을 가정한 데모 제작
3. 조위·기상·유동인구·사고이력의 결합 예시를 시각화
4. 모델은 실제 운영 전 `PoC → 시범구간 → 확대` 단계로 제시

## 절대 과장하지 말 것

- “AI가 사고를 완전히 막는다” 금지
- “개인을 찾아내 경고한다” 금지
- “실시간 구조 시스템”으로 포장 금지
- “출입통제를 자동화한다” 금지

권장 표현:

- 사고 위험 노출을 줄인다
- 예방 행정 판단을 지원한다
- 고위험 시간·장소·상황을 조기에 식별한다
- 관계기관 협업을 위한 근거 데이터를 제공한다
