# MVP 명세

## MVP 목표

공모전 제출 및 발표에서 “이 서비스가 실제로 어떻게 보이는지”를 선명하게 보여주는 최소 제품을 정의한다.

MVP는 실제 운영 시스템 전체가 아니라, **안산시 담당자가 위험을 예측하고 예방 조치를 선택하는 흐름**을 보여주는 데 집중한다.

## MVP 핵심 메시지

> 안산시는 사고가 난 뒤 알게 되는 것이 아니라, 위험한 시간이 오기 전에 미리 볼 수 있다.

## MVP 사용자

- 안산시 스마트도시과/재난안전/관광 관련 담당자
- 현장 관리 담당자
- 관계기관 협업 담당자

## MVP 화면 구성

### 화면 1. 오늘의 안산 연안 위험지도

목적:

- 심사위원에게 서비스의 본체를 직관적으로 보여준다.

요소:

- 안산 연안 지도
- 위험지점 마커
- 위험도 색상: 낮음/주의/높음/긴급
- 시간 슬라이더
- 필터: 조위, 기상, 유동인구, 사고이력, 혼잡

예시 문구:

```text
2026.07.04 토요일 17:00 기준
방아머리 인근 위험도 82점 / 높음
위험 상승 원인: 만조 임박, 유동인구 증가, 강풍 예보, 과거 유사 사고 이력
```

### 화면 2. 위험도 상세 패널

목적:

- AI 판단의 설명가능성을 보여준다.

요소:

- 위험도 점수
- 주요 원인 Top 5
- 시간대별 위험도 그래프
- 과거 유사 패턴
- 관련 데이터 출처

예시:

```text
위험도: 82점
주요 원인:
1. 만조까지 47분
2. 유동인구 평소 대비 165%
3. 풍속 상승
4. 일몰 시간대 진입
5. 과거 유사 조건에서 사고 발생 이력
```

### 화면 3. 예방 개입 추천 카드

목적:

- 앱이 아니라 행정 의사결정 시스템임을 보여준다.

요소:

- 추천 조치
- 실행 주체
- 권한 구분
- 기대효과
- 실행 난이도
- 관계기관 공유 필요 여부

예시 카드:

```text
추천 조치 A: 전광판 문구 변경
실행 주체: 안산시 현장 관리 부서
권한 위험: 낮음
예상 효과: 위험시간대 갯벌 체류 감소
권장 문구: “조위 상승 중입니다. 갯벌 체류자는 즉시 안전지대로 이동하세요.”
```

```text
추천 조치 B: 관계기관 공유 리포트 발송
실행 주체: 안산시 → 해경/소방 등 관계기관
권한 위험: 낮음
예상 효과: 고위험 시간대 사전 상황 공유
```

### 화면 4. 익명 상황인식 패널

목적:

- 개인 감시가 아니라 비식별 집계 기반임을 보여준다.

요소:

- 구역별 추정 인원 수
- 이동 방향 화살표
- 체류 시간 증가 지표
- 위험구역 접근량
- 원본 영상 미표시 또는 블러/비식별 표시

주의:

- 사람 얼굴, 실제 개인 추적 UI처럼 보이는 이미지는 넣지 않는다.
- “개인별 위험도” UI는 만들지 않는다.

### 화면 5. 개입 효과 리포트

목적:

- 효과성과 정책 확장성을 보여준다.

요소:

- 개입 전후 혼잡도 변화
- 위험구역 체류 변화
- 신고/민원 변화
- 다음 주말 추천 전략
- 시설 개선 우선순위

예시:

```text
지난 4주간 고위험 시간대 전광판 안내 후 위험구역 체류 추정 인원이 평균 18% 감소했습니다.
다음 개선 후보: A구역 안내판 위치 조정, B구역 야간 조명 보강.
```

## MVP 기능 목록

| 우선순위 | 기능 | 설명 |
|---:|---|---|
| P0 | 위험지도 | 장소·시간별 위험도 시각화 |
| P0 | 위험도 산출 | 조위·기상·유동인구·사고이력 결합 |
| P0 | 원인 설명 | 위험 상승 이유 Top 3 표시 |
| P0 | 개입 추천 | 안산시가 할 수 있는 예방 조치 추천 |
| P1 | 관계기관 공유 리포트 | 해경·소방 등 공유용 요약 생성 |
| P1 | 익명 현장 신호 | 사람 수·방향·체류시간 집계 표시 |
| P1 | 효과 리포트 | 개입 전후 변화 분석 |
| P2 | 알림 채널 연동 | 전광판, 관광안내, 문자 등 연동 |
| P2 | 시설 개선 제안 | 반복 위험구간 기반 개선 후보 추천 |

## MVP에서 제외할 기능

- 얼굴인식
- 개인별 위험점수
- 자동 출입통제
- 구조 지휘
- 시민 제재
- 복잡한 모바일 앱 전체 구현
- 모든 지역 실시간 관제

## 데모 데이터 전략

실제 API 연동이 어렵더라도 데모는 가능해야 한다.

### 샘플 데이터 테이블

```csv
datetime,location_id,tide_level,tide_change,wind_speed,rainfall,visitor_index,accident_history_score,risk_score
2026-07-04 17:00,A01,high,rising,8.2,0,165,70,82
2026-07-04 18:00,A01,high,rising,9.1,0,172,70,88
2026-07-04 19:00,A01,peak,falling,8.7,0,154,70,79
```

### 산출 공식 예시

초기 MVP에서는 복잡한 모델 대신 설명 가능한 가중합 또는 간단한 ML 모델로 시작해도 된다.

```text
risk_score = f(조위위험, 기상위험, 유동인구, 사고이력, 야간/일몰, 현장혼잡)
```

발표에서는 “운영 초기에는 설명 가능한 모델로 시작하고, 데이터가 쌓이면 시공간 예측 모델로 고도화한다”고 설명한다.

## 발표용 30초 시연 흐름

1. 지도에서 오늘 오후 고위험 구간이 붉게 표시된다.
2. 클릭하면 위험 원인 Top 3가 나온다.
3. 시스템이 전광판 문구 변경, 관광안내 알림, 관계기관 공유를 추천한다.
4. 담당자가 조치를 선택하면 공유 리포트가 생성된다.
5. 다음 날 효과 리포트에서 위험구역 체류 감소가 표시된다.

## 성공 기준

MVP가 성공하려면 심사위원이 다음 세 가지를 바로 이해해야 한다.

1. 왜 안산시에 필요한가?
2. 왜 공공데이터와 AI가 필요한가?
3. 왜 단순 앱/챗봇보다 발전된 아이디어인가?
